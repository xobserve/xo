Datav的侧边菜单栏上半部分是可以自定义的，这样就可以实现自定义网站的效果。

其中，每个团队都可以定制自己的侧边菜单栏，团队的自定义菜单栏默认是私有的，但你可以选择公开化，这样每个人都能使用。

可能大家会有疑问，这样的好处是什么？ 想象一下，我们现在要做一个监控平台，有两个监控大类：应用监控和基础设施监控，其中应用监控是开发关注的，
基础设施监控是运维关注的，那么就可以为运维团队专门创建一个侧边菜单，当运维使用Datav的时候默认看到的就是基础设施相关的菜单项，同时运维也可以切换到
应用监控的菜单项，来查看应用监控内容。

当公司规模较大时，自定义菜单会带来极大的便利，只需要部署管理一套Datav即可。

## 菜单项面板
1. 选择**配置管理 -> 团队 -> 查看global -> 侧边菜单**，这里能看到菜单的ID、简介、是否可公开，ID是方便后期在url地址栏快速访问的，简介可以在菜单公开化后，让其它用户更清晰的看到菜单的用途
2. 在菜单管理中，已经有了默认的菜单项`Home`，其实大家都已经访问过这个菜单项了，就是侧边菜单最顶部的图标，**点击Home Dashboard所在的行**
3. 现在弹出了两个面板，左边是更新当前的菜单项，右边是在当前菜单项旁边或者下面创建新的菜单项，这里我们先从更新菜单项开始

## 更新菜单项
在左边面板我们可以更新当前菜单项，仪表盘uid是每个仪表盘独有的字符串id类型，可以自定义修改，通过这里的设置，可以把菜单项跟我们的仪表盘一对一关联起来，默认情况下，当前的菜单项
关联到了`home`仪表盘。

Icon名称是用来设定菜单项图标的，具体的图标请在[Unicons](https://iconscout.com/unicons/explore/line)网站进行查找。

菜单项名称是侧边菜单中显示的名称，该名称可以和仪表盘自身的名称不同。

菜单url指定了该菜单项被访问时的url，例如：当前设置是/home，那么当用户点击该菜单项时，会跳转到http://localhost:3001/home进行访问。
<TipCompat>注意，该url不能重复，且不能使用datav已经预占用的url</TipCompat>。

背景知识介绍完毕，下面来更新下当前菜单项:
1. 仪表盘uid不要改，继续关联到之前的首页仪表盘
2. Icon名称修改为`apps`
3. 菜单项名称修改为`应用监控`
4. 菜单url修改为`/apps`
5. 点击提交按钮
6. 点击更新按钮

现在刷新页面，然后就可以看到侧边菜单的菜单项已经改变,把鼠标悬浮上去，然后访问查看效果。


## 添加菜单项
继续进入菜单项配置面板，现在聚焦到右边的面板**在菜单项`应用监控`旁边添加新菜单项**，因为我们要创建`应用监控`的二级菜单，因此`添加到以下位置`选择`子节点`
1. 使用o s(先按o，再按s)打开搜索面板，把鼠标悬浮到之前章节我们创建的仪表盘上，可以看到`dashboard uid: XXXX`的提示，把XXXX复制下来
2. 按`esc`关闭搜索面板，将刚才复制的uid粘贴到`仪表盘uid`下
3. Icon名称输入`analytics`
4. 菜单项名称输入`应用分析`
5. 这里的菜单url是相对的，如果是子节点，就输入当前的二级url即可，不需要加上父url，这里输入`/analytics`
6. 提交 & 更新

刷新查看效果

## 调整菜单项顺序
在菜单管理中，可以拖动**菜单所在的行**来跟其它菜单交换位置

至此，我们已经为global团队成功修改了菜单，下一步来看看用户该怎么使用这些菜单

## 使用菜单
选择侧边菜单**用户 -> 偏好设置**，在高级设置里就能选择菜单了，切记，只能选择以下的菜单:
1. 所在团队的菜单
2. 其它团队公开的菜单


## 通过url设置菜单
为了让用户更灵活的使用菜单，Datav支持通过url来设置菜单，还记得global团队的菜单id吧？如果不记得，再回头去看一下：）

现在在浏览器地址栏添加上以下字符串`?sidemenu=2`，访问试试看，如果报错，就对了，因为我们还没有创建其它菜单，也就是id为`2`的菜单。

现在再访问`?sidemenu=1`，一切正常，因为global团队的菜单id是`1`。
